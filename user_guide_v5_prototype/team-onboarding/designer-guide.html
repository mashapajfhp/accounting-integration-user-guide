üîç User Research Insights

üí° Key User Insights from JIRA Analysis (85+ Customer Issues)

Real customer behavior patterns inform our design decisions:

Wizard Navigation Issues

Problem: Users frequently get stuck on disabled "Next" buttons

Data: JIRA tickets OS-2065, TSSD-2802, TSSD-1050

User Impact: Frustration and setup abandonment

Design Solution: Enhanced validation feedback and progress indicators

Error Message Clarity

Problem: Backend IDs in error messages confuse users

Data: JIRA ticket TSSD-2552 (recently improved)

User Impact: Cannot self-resolve issues

Design Solution: Plain language error messages with actionable guidance

Loading State Anxiety

Problem: Long connection times (3+ hours) create user anxiety

Data: JIRA tickets TSSD-2441, GS-5269

User Impact: Users assume system is broken

Design Solution: Expectation setting and progress communication

Page Refresh Requirements

Problem: Some configurations require manual page refresh

Data: JIRA ticket GS-5373

User Impact: Broken user expectations of SPA behavior

Design Solution: Auto-refresh or clear refresh guidance

Consent-first design: Clear permission explanation builds user trust

üß™ User Behavior Patterns Discovered

  • Dual Discovery Pattern: Users find feature through Settings (planned) or Payroll (contextual)
  • Permission Hesitancy: Users need reassurance about data access scope
  • Platform Familiarity: Users more confident with accounting systems they know
  • Validation Dependency: Users want to preview before committing to changes

    üìà Design Evolution & User Feedback

    üîÑ Design Iterations Based on Real Customer Feedback

    How user research and support tickets shaped the current design:

    Iteration 1: Wizard Navigation

    Problem Identified: Users couldn't progress through configuration (JIRA: OS-2065)

    Design Change: Enhanced validation feedback and disabled state explanations

    Outcome: Reduced navigation-related support tickets

    Learning: Complex forms need explicit validation communication

    Iteration 2: Error Messaging

    Problem Identified: Backend error IDs confused users (JIRA: TSSD-2552)

    Design Change: Plain language error messages with actionable guidance

    Outcome: Improved user self-resolution capabilities

    Learning: Technical errors need human-friendly translation

    Iteration 3: Loading States

    Problem Identified: Long connection times create anxiety (JIRA: TSSD-2441)

    Design Change: Time expectation setting and progress communication

    Outcome: Reduced user abandonment during setup

    Learning: Manage user expectations for long processes

    Iteration 4: State Management

    Problem Identified: Manual refresh required for updates (JIRA: GS-5373)

    Design Challenge: SPA behavior expectations vs system limitations

    Current Workaround: Clear refresh guidance where needed

    Future Solution: Real-time state synchronization

    Post-configuration management: Clean action hierarchy with clear consequences
