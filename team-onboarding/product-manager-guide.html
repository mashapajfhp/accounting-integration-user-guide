Architecture Decision: Partner with Codat for unified API access rather than building direct integrations.

  • Faster time-to-market for multiple platforms
  • Reduced maintenance overhead
  • Leveraged authentication and data mapping
  • Platform constraint inheritance (limitation)

üìã Top Customer Requests (Based on JIRA Analysis)

Understanding these requests helps prioritize roadmap and manage customer expectations:

Multi-Dimensional Cost Centers

Customer Need: Enterprise clients need to split employee costs across multiple business dimensions simultaneously.

Example: 60% Project Alpha + Dubai Office vs 40% Project Beta + Abu Dhabi Office

Current Limitation: Single dimension per employee only

Business Impact: Blocks enterprise deals requiring complex cost allocation

Source: JIRA Analysis - TSSD-3194, TSSD-3188

Platform Feature Gaps

Challenge: Not all enterprise features available on all platforms

  • D365 BC: Cost centers not supported
  • Xero: Tracking category limitations
  • NetSuite: Complex permission requirements
  • QuickBooks: Split amount validation issues

Impact: Platform choice affects feature availability for customers

Support Complexity

Reality: Integration issues require both accounting expertise and technical knowledge

Support Challenge: Complex troubleshooting needs specialized team

Cost Impact: Higher support costs vs standard HR features

Training Need: Support team needs accounting software knowledge

Implementation Reality

Setup Complexity: 23% of customers need support assistance

  • D365: Connection can take 3+ hours
  • NetSuite: Multiple permission requirements
  • All platforms: Wizard navigation challenges

Impact: Implementation friction affects customer experience

Real example: Configuration complexity causing 23% of support tickets
